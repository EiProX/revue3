Règle métier

Du pointage brut à la session

Un pointage est un événement simple: arrivée ou départ. Le backend reconstruit ensuite les sessions de présence avec une heure d'arrivée, une heure de départ et une durée en minutes.

Code à expliquer

Route de pointage manuel

Cet extrait montre une route Express classique: on protège l'accès, on vérifie les données reçues, puis on appelle le service qui écrit en base. 

router.post("/api/manual/now", requireAuth, async (req, res) => {
  // 1. On récupère les informations envoyées par l'interface.
  const { id_etudiant, action } = req.body;

  // 2. On évite d'écrire en base si la requête est incomplète.
  if (!id_etudiant || !action) {
    return res.status(400).json({ error: "Champs manquants" });
  }

  // 3. Seules deux actions sont autorisées: ARRIVEE ou DEPART.
  if (!ALLOWED_POINTAGE_TYPES.includes(action)) {
    return res.status(400).json({ error: "action invalide" });
  }

  // 4. Le service crée le pointage et resynchronise la journée.
  const result = await createManualPointage({ idEtudiant: id_etudiant, action });

  res.json({ ok: true, date_jour: result.date_jour });
});

Code à expliquer

Créer un pointage puis mettre à jour les sessions

On crée un événement de pointage, puis on demande au backend de recalculer les sessions de la journée. 

async function createManualPointage({ idEtudiant, action }) {
  // 1. On récupère la date du jour et l'heure actuelle.
  const date = todayISO();
  const horodatage = nowParisDateTimeSQL();

  // 2. On enregistre le pointage brut dans la table "pointages".
  await insertPointage({
    idEtudiant,
    typePointage: action,    // ARRIVEE ou DEPART
    horodatage,
    modeSaisie: "MANUEL"
  });

  // 3. Le backend recalcule les sessions à partir des pointages.
  // Exemple: une ARRIVEE + un DEPART deviennent une session avec durée.
  await syncPointagesForDate(date);

  // 4. La route peut répondre à l'interface avec la date traitée.
  return {
    date_jour: date,
    horodatage
  };
}

Sécurité et traces

Deux éléments: une route protégée refuse un utilisateur non connecté, et chaque action importante peut être journalisée. 

function requireAuth(req, res, next) {
  // Si aucune session utilisateur n'existe,
  // l'API refuse l'accès à la route.
  if (!req.session.user) {
    return res.status(401).json({ error: "Non authentifié" });
  }

  next();
}
